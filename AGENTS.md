# AGENTS.md

## Learned User Preferences

- Dutch-speaking user; respond in Dutch when they write in Dutch. App UI is English-only (overlay placeholders, status strings, error messages).
- Target model picker selects guide methodology only; settings expose a single OpenAI API key; one rewrite model (GPT-4.1 mini) handles all generation.
- Ctrl+Shift+O overlay must auto-populate from the active Windows text field — including when typing inside PromptForge Studio, not only external apps like Cursor/Chrome; manual paste is not acceptable. Apply must inject back into the captured field (Cursor/Chrome/native); on inject failure refined text must stay on the clipboard for manual paste — never restore the pre-capture clipboard over the fallback copy.
- Overlay does not auto-optimize on open; Refine button (or Enter when idle) triggers the rewrite API.
- Rejected clean-only conservative rewrite; refinement must follow model-specific prompting-guides/ with level-specific structure contracts (Cool plain prose through Max full XML).
- Prefer clear API errors over silent local/mock fallback when rewrite fails — no substitute rewrite output on missing key or provider errors.
- Overlay output should be plain refined prompt text only — no JSON, no rubric/score UI in the overlay; after generation the output textarea is editable and Apply/Copy use the edited text; Generate becomes Regenerate after the first run in a session (resets on new capture).
- PromptForge optimizes prompts for the target LLM; launch is overlay/tray-only until user opens Studio via Settings or tray; RAM/geheugenoptimalisatie mag UX niet wijzigen — overlay hotkey, capture, Refine en Apply blijven identiek.
- Overlay opens on the active display work area using the user's snap placement (default center); no black scrim/backdrop; no open animation and no flashy flicker masks — speed and subtle modern look prioritized. Hotkey UX: glass shell appears instantly on hotkey (`showOverlayShell`); captured text fills in shortly after; no `OVERLAY_CAPTURE_PENDING` spinner path; instant feel preferred; first hotkey after reload is gold-standard for every subsequent open.
- Overlay position: five snap placements (corners + center) chosen via monitor skeleton in overlay ⋮-menu; `overlayPlacement` in AppSettings persisted; applied on hotkey open and immediately when changed in menu — no free drag.
- Overlay shell is CSS glassmorphism on `.apple-glass` only — no React Bits `GlassSurface`, no Windows `setBackgroundMaterial('acrylic')` on the full overlay window (causes large white box); inner `apple-glass-panel`/`apple-glass-pill`/`apple-glass-menu` unchanged; pill buttons less transparent than shell.
- Overlay footer UX: L1–L4 draggable level slider beside Generate — thin grey track, colorful Cool/Warm/Hot/Max label, spring snap; Discard next to Apply/Copy in light gray (`text-white/45`), not pink; terminal sessions use the same footer (Apply injects, not copy-only). Terminal rewrite output must be one line — no Enter/newlines (shell paste breaks); overlay blocks Enter in output textarea.
- Overlay folder tabs (user's 2026-07-07 concept, `.cursor/plans/modal UI outline concept.md`): four `.apple-glass-tab` tabs tucked BEHIND the card (`z-0` under card `z-10`, near-opaque gradient — never translucent overlap, never a seamless border merge). Top tab = PromptTypePicker (Auto/Question/Prompt/Letter, width-measured select, default "auto", resets per capture session); bottom-left tab = ModelPicker (moved out of the footer, chevron ~7px width-measured rule still applies); left tab = vertical "PromptForge" wordmark (decorative, `aria-hidden`); right tab = ⋮ menu bump (menu itself is a sibling at `z-20`, NOT nested in the tab — a child of the z-0 tab would render under the card). Non-auto promptType adds a PROMPT TYPE rule to `buildMetaPrompt` (terminal single-line rule still overrides) and suffixes the opt-cache hash with `|type:{t}`; "auto" adds nothing and keeps old cache entries valid (no REWRITE_PIPELINE_VERSION bump).

## Learned Workspace Facts

- PromptForge: Windows-only Electron + Vite + React/TypeScript; npm scripts `dev`, `build`, `test` (vitest), `lint`; architecture in `src/main`, `src/renderer`, `src/engine`, `src/shared`. Dev bundler: `vite.config.mts` with `@vitejs/plugin-react-swc`, `css.devSourcemap: false`, `hmr.overlay: false`; Studio lazy-loaded in `router.tsx` (separate chunk).
- Rewrite always uses `REWRITE_CONFIG` (OpenAI `gpt-4.1-mini`); target model picker selects which `prompting-guides/` file `guideLoader` loads.
- `prompting-guides/` source of truth per ModelId: `opus4.8.md`, `gpt5.5.md`, `gemini.3.pro.md`, `deepseek.V3.md`, `grok4.md`, `composer2.5.md`.
- L1–L4 = guide structure adherence scale (Cool/Warm/Hot/Max); knob sets guide excerpt + rewrite intensity. `getLevelStructureContract()` in `guideLoader.ts` fixes output shape: Cool=plain prose; Warm=`<instructions>`+`<input>`; Hot=`<context>`/`<task>`/`<constraints>`/`<output_format>`; Max=Hot plus `<examples>`/`<success_criteria>` (must retain all L3 tags). `buildMetaPrompt` in `providers.ts` embeds contract plus action/constraint framing from L2+. `REWRITE_PIPELINE_VERSION` in cache hash; Regenerate sends `skipCache: true`. Temperature fixed at `REWRITE_CONFIG.temperature` (0.3); measured adherence via `adherenceLevel()` in `rubric.ts`.
- Launch: tray-only — no Studio window on dev/launch (removed `ensureStudio()` from startup and `app.on("activate")`); overlay on Ctrl+Shift+O; Studio opens only via the overlay Settings button or tray, both routed through `STUDIO_SETTINGS`/`ensureStudio("settings")` (tray Settings was previously broken — `webContents.send` lacked a preload bridge).
- Hotkey flow: `prepareCaptureTarget()` → `showOverlayShell()` (instant glass, empty field, `setOpacity(1)`) → `hotkeySnapshot()` → fast path skips `hideForCapture()` → `captureSelection()` → `deliverCaptureToOverlay()` fills captured text (second `OVERLAY_SHOW`; no `OVERLAY_CAPTURE_PENDING` on hotkey). **Do not reorder to capture-before-shell** — that delays visible popup until PS capture completes even on fast path (regressed in a later session). **Blur hide must skip while `hotkeyInFlight`** — otherwise `showOverlayShell` + snapshot `FocusWindow` closes the overlay mid-capture and popup reappears only after `deliverCaptureToOverlay` (feels slow). Resident overlay: `hideOverlay()` sends `OVERLAY_CLEAR`, awaits `OVERLAY_PREPARED`, sets opacity 0 — does not `hide()`; `overlaySessionOpen` drives blur/toggle (not `isVisible()`). `revealOverlay()` idempotent: `setOpacity(0)` → `showInactive()` only if not visible. `primeOverlayBuffer()` at startup leaves window warm via `showInactive()`. Overlay `backgroundThrottling: false`; renderer `ackOverlayPrepared()` 1× rAF then main `setOpacity(1)` on shell reveal (Windows transparent-window white-flash guard only, not an open animation).
- PS script shared types: `terminal-io.ps1` OWNS the shared `WinFg` C# class; scripts that dot-source it (`win-hotkey-snapshot.ps1`, `win-inject.ps1`) must NOT define their own `WinFg` unguarded — a duplicate `Add-Type` throws `TYPE_ALREADY_EXISTS` under `$ErrorActionPreference='Stop'`, which crashes `hotkeySnapshot` and forces the slow `win-capture.ps1` fallback (laggy popup that appears then reopens). `win-hotkey-snapshot.ps1` keeps its local `WinFg` behind `if (-not ([PSTypeName]'WinFg').Type)`. `win-inject.ps1` uses `WinInject`, `win-capture.ps1` uses `WinCapture` (distinct names, no collision). `WinFg.GetWindowThreadProcessId` P/Invoke must be `out uint lpdwProcessId` (not `IntPtr`); never use `$pid` as out var — PowerShell `$PID` is read-only; use `$winPid`.
- Capture/inject via PS scripts (`win-capture.ps1`, `win-hotkey-snapshot.ps1`, `win-inject.ps1`); foreground HWND polled every 150ms via koffi `src/main/win32.ts` (`GetForegroundWindow`, not `get-foreground.ps1` per tick); `pollInFlight` mutex; `ps-warmup.ps1` + `warmCaptureBridge()` at launch reduce first-hotkey PS/UIA cold start. Shared terminal PS helpers in `scripts/terminal-io.ps1` (`Resolve-TerminalPaneElement`, `Focus-TerminalPane`, keyboard/clipboard). Terminal hosts use `CaptureMode` `"terminal"`: `win-hotkey-snapshot.ps1` reads UIA `TextPattern.GetSelection()` then clipboard fallback — Ctrl+Shift+C for Windows Terminal, Ctrl+C for conhost (`ConsoleWindowClass`); clipboard restored; never `win-capture.ps1` keystroke fallbacks on terminals. Frozen `terminalBounds` from capture flows to inject when UIA resolve fails. `terminalDetect.ts` detects native class/process plus Cursor/VS Code integrated panes (`focusedIsTerminalPane`); filters a11y noise/window-title leaks. All paths normalize via `resolveCaptureFromPossibleTerminal()`/`extractTerminalInput()` in `terminalCapture.ts` — keeps scrollback output + current prompt line, strips banners/a11y noise.
- Apply: `injectText()` via `win-inject.ps1` using frozen target from capture (`injectStrategy.ts` hostKind native/chromium/richEditor/terminal; `scripts/inject-strategy.ps1` in PS scripts); overlay label "Apply" for all hosts including terminal. Field strategy: one focus-restore → resolve element → host-specific primary+fallback (native: ValuePattern→clipboard paste→unicode; chromium/richEditor: clipboard paste→unicode) — not the old 5-method cascade. Terminal: `freezeTerminalInjectTarget()` on capture; same IPC path as fields (no `main.ts` terminal clipboard short-circuit); `Try-TerminalPaste` = single focus+select+paste with optimistic verify — right-click paste WT/conhost, Ctrl+V integrated; no strategy waterfall or clipboard readback verify cascade. `main.ts` `waitUntilForeground()` + ~200ms settle before inject; failure `revealOverlayAfterInjectFallback()` keeps session, success `finalizeOverlayAfterInject()`. Clipboard fallback on inject failure keeps refined text; restore pre-capture snapshot only after verified inject (~200ms). Ctrl+A/V via `keybd_event` (SendInput unreliable for combos from PS x64); Unicode typing still SendInput. Chromium/richEditor: optimistic verify when UIA readback fails after paste. `terminalContext` on optimize → TERMINAL SHELL meta-prompt (single-line plain output overrides L2+ tags); `toTerminalSingleLine()`/`stripTerminalStreamChunk()` in `terminalOutput.ts` on stream/Apply/inject; cache hash includes `|terminal`. PS scripts read fresh per hotkey — script edits need no rebuild; TS changes need `npm test`/dev reload. Overlay "Original" is editable `<textarea>`; readOnly while busy/`"capturing"`; Enter inserts newline (does not trigger Refine). Preload registers `OVERLAY_SHOW`/`onOverlayCapturePending` with replay; `IPC.OVERLAY_CLEAR` on hide; overlay created hidden; `deliverCaptureToOverlay()` waits for `did-finish-load` if needed.
- Dev/memory: `optCache` LRU cap 100 (slim entries, no persisted `diff`); `spellcheck: false`; overlay `backgroundThrottling: false`, Studio `true`; Studio nulled on `closed`; no React StrictMode in dev. Do not auto-open detached DevTools (steals focus, breaks capture); browser UI at `localhost:5173#/overlay-preview` only. Vite proxies `/api/optimize` + `/api/settings` to `127.0.0.1:5174` — never all `/api`. Capture/inject no-op in browser. PS shell: chain with `;`, no `&&`/heredocs.
- API keys stored encrypted via OS Credential Manager (`keyStore`); `orchestrator.ts` requires key and throws on missing key, API failure, or empty rewrite — no `optimizeLocal` fallback; streams chunks live to overlay/Studio (both show optimize errors). `localOptimizer.ts` kept for unit tests. Overlay output is plain refined prompt text (`stripResponseArtifacts`). UIA snapshot metadata stays local for Apply — only prompt text goes to the rewrite API.
- Overlay UI: `positionOverlay()` + renderer `items-center justify-center`; transparent window (no scrim), snap placement via `OVERLAY_PLACEMENT_SET` IPC, `overlayPlacement` in settings. ⋮-menu (right folder tab) includes `OverlayPlacementPicker` skeleton, Settings, Open in Studio, Dismiss. Shell is `div.apple-glass` in `Overlay.tsx` — layered CSS gradients (~90% alpha) with `::before`/`::after` highlights in `index.css`; `GlassSurface` removed. CSS `backdrop-filter` cannot blur the OS desktop through a frameless transparent Electron window on Windows — frosted look is simulated via opaque gradients/tint only; do not use acrylic on the overlay window. Output uses `useTypewriterReveal` during Generate/Regenerate (~3 chars/frame with catch-up; `prefers-reduced-motion` skips animation) and native `scroll-thin` scrollbar. `ModelPicker` measures label width for chevron placement; `LevelSlider` beside Generate; footer order Discard → Apply → Copy for all modes. `terminalContext` empty capture shows "Select text in the terminal first, then press the hotkey."
