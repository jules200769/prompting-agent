# AGENTS.md

## Learned User Preferences

- Dutch-speaking user; respond in Dutch when they write in Dutch. App UI is English-only (overlay placeholders, status strings, error messages).
- Target model picker selects guide methodology only; settings expose a single OpenAI API key; one rewrite model (GPT-4.1 mini) handles all generation.
- Ctrl+Shift+O overlay must auto-populate from the active Windows text field — including when typing inside PromptForge Studio, not only external apps like Cursor/Chrome; manual paste is not acceptable.
- Overlay does not auto-optimize on open; Refine button (or Enter when idle) triggers the rewrite API.
- Rejected clean-only conservative rewrite; refinement must follow model-specific prompting-guides/ with level-specific structure contracts (Cool plain prose through Max full XML).
- Prefer clear API errors over silent local/mock fallback when rewrite fails — no substitute rewrite output on missing key or provider errors.
- Overlay output should be plain refined prompt text only — no JSON, no rubric/score UI in the overlay; after generation the output textarea is editable and Apply/Copy use the edited text; Generate becomes Regenerate after the first run in a session (resets on new capture).
- PromptForge optimizes prompts for the target LLM; launch is overlay/tray-only until user opens Studio via Settings or tray; RAM/geheugenoptimalisatie mag UX niet wijzigen — overlay hotkey, capture, Refine en Apply blijven identiek.
- Overlay opens centered on the active display work area; no black scrim/backdrop; no open animation and no flashy flicker masks — speed and subtle modern look prioritized. Hotkey UX: one clean reveal per hotkey with final text (no Capturing→text swap on fast path); instant feel preferred, ~50–150ms pre-reveal delay acceptable; first hotkey after reload is gold-standard for every subsequent open.
- Overlay must be freely draggable like a native Windows window (no movement restrictions) and remember its position across reopens.
- Overlay shell is CSS glassmorphism on `.apple-glass` only — no React Bits `GlassSurface`, no Windows `setBackgroundMaterial('acrylic')` on the full overlay window (causes large white box); inner `apple-glass-panel`/`apple-glass-pill`/`apple-glass-menu` unchanged; pill buttons less transparent than shell.
- Overlay footer UX: ModelPicker chevron tight beside selected label (~7px gap, width-measured); L1–L4 draggable level slider beside Generate — thin grey track, colorful Cool/Warm/Hot/Max label, spring snap; Discard next to Apply/Copy in light gray (`text-white/45`), not pink; terminal capture footer is Discard → single Copy pill only (secondary Copy hidden; copy-only, never inject back to shell).

## Learned Workspace Facts

- PromptForge: Windows-only Electron + Vite + React/TypeScript; npm scripts `dev`, `build`, `test` (vitest), `lint`; architecture in `src/main`, `src/renderer`, `src/engine`, `src/shared`. Dev bundler: `vite.config.mts` with `@vitejs/plugin-react-swc`, `css.devSourcemap: false`, `hmr.overlay: false`; Studio lazy-loaded in `router.tsx` (separate chunk).
- Rewrite always uses `REWRITE_CONFIG` (OpenAI `gpt-4.1-mini`); target model picker selects which `prompting-guides/` file `guideLoader` loads.
- `prompting-guides/` source of truth per ModelId: `opus4.8.md`, `gpt5.5.md`, `gemini.3.pro.md`, `deepseek.V3.md`, `grok4.md`, `composer2.5.md`.
- L1–L4 = guide structure adherence scale (Cool/Warm/Hot/Max); knob sets guide excerpt + rewrite intensity. `getLevelStructureContract()` in `guideLoader.ts` fixes output shape: Cool=plain prose; Warm=`<instructions>`+`<input>`; Hot=`<context>`/`<task>`/`<constraints>`/`<output_format>`; Max=Hot plus `<examples>`/`<success_criteria>` (must retain all L3 tags). `buildMetaPrompt` in `providers.ts` embeds contract plus action/constraint framing from L2+. `REWRITE_PIPELINE_VERSION` in cache hash; Regenerate sends `skipCache: true`. Temperature fixed at `REWRITE_CONFIG.temperature` (0.3); measured adherence via `adherenceLevel()` in `rubric.ts`.
- Launch: tray-only — no Studio window on dev/launch (removed `ensureStudio()` from startup and `app.on("activate")`); overlay on Ctrl+Shift+O; Studio opens only via the overlay Settings button or tray, both routed through `STUDIO_SETTINGS`/`ensureStudio("settings")` (tray Settings was previously broken — `webContents.send` lacked a preload bridge).
- Hotkey flow: `hotkeySnapshot()` → fast path skips `hideForCapture()` → `captureSelection()` → one `deliverCaptureToOverlay()` with final text (`OVERLAY_SHOW` then `revealOverlay()`; no `OVERLAY_CAPTURE_PENDING` on hotkey). `revealOverlay()` idempotent: `setOpacity(0)` → `showInactive()`. `hideOverlay()` sends `OVERLAY_CLEAR`, awaits `OVERLAY_PREPARED` (`flushSync`, `shellVisible` false), then `hide()` — opacity stays 0 until next reveal (stale-frame fix for 2nd+ hotkey). `primeOverlayBuffer()` at startup. Overlay `backgroundThrottling: false`; after prepared, ~16ms tick then `setOpacity(1)` (Windows transparent-window white-flash only, not an open animation).
- Capture/inject via PS scripts (`win-capture.ps1`, `win-hotkey-snapshot.ps1`, `win-inject.ps1`); foreground HWND polled every 150ms via koffi `src/main/win32.ts` (`GetForegroundWindow`, not `get-foreground.ps1` per tick); `pollInFlight` mutex; `ps-warmup.ps1` + `warmCaptureBridge()` at launch reduce first-hotkey PS/UIA cold start. Terminal hosts use `CaptureMode` `"terminal"`: `win-hotkey-snapshot.ps1` reads UIA `TextPattern.GetSelection()` then clipboard fallback — Ctrl+Shift+C for Windows Terminal, Ctrl+C for conhost (`ConsoleWindowClass`); clipboard restored; never `win-capture.ps1` keystroke fallbacks on terminals. `terminalDetect.ts` detects native class/process plus Cursor/VS Code integrated panes (`focusedIsTerminalPane`); filters a11y noise. All paths normalize via `resolveCaptureFromPossibleTerminal()`/`extractTerminalInput()` in `terminalCapture.ts` (PowerShell banner/buffer → prompt-line input).
- Apply: `injectText()` replaces field text using saved UIA metadata; overlay button label is "Apply" (not "Inject"); terminal mode relabels primary pill to "Copy" and routes Apply/`CAPTURE_INJECT` to clipboard only — no `injectText()`/shell inject. `win-inject.ps1` inject methods cascade ValuePattern → TextPattern → Unicode → Keyboard (each its own focus-click + select-all = visible "too many steps"); Chromium hosts (top-level class `Chrome_WidgetWin` covers Chrome/Edge/Brave AND Electron Cursor/ChatGPT desktop, detected via `AutomationElement.FromHandle($top).Current.ClassName` → `$script:IsChromiumHost`) force `Test-IsRichTextEditor` true so all web fields use the single `Try-RichEditorPaste` path (focus → Ctrl+A → Ctrl+V → verify); native Win32 (Notepad) keep the ValuePattern path. PS scripts read fresh per hotkey — script edits need no rebuild. Overlay "Original" is an editable `<textarea>` bound to `prompt`; readOnly while busy or during `"capturing"`; Enter inserts newline (does not trigger Refine). Preload registers `OVERLAY_SHOW` and `onOverlayCapturePending` before React mount, each with replay; `IPC.OVERLAY_CLEAR` on hide clears replay buffers; overlay created hidden at startup; `deliverCaptureToOverlay()` waits for `did-finish-load` if needed. No `.overlay-in` or `key` remount on IPC; `shellVisible` keeps card invisible until capture applied; optional subtle input-panel ring/pulse during capture.
- Memory hygiene: `optCache` LRU cap 100 with slim entries (no persisted `diff`; rebuilt via `buildDiff` on read); `spellcheck: false` on all windows; overlay `backgroundThrottling: false`, Studio `backgroundThrottling: true`; Studio window nulled on `closed`; React StrictMode removed in dev renderer.
- Dev gotchas: do not auto-open detached DevTools (steals focus, breaks capture); plain browser at `localhost:5173` — open `#/overlay-preview` for overlay UI review (Electron hotkey overlay not agent-controllable). With `npm run dev`, browser optimize/settings proxy via Vite to Electron dev bridge on `127.0.0.1:5174` (`/api/optimize`, `/api/settings` only — never proxy all `/api` or `api.ts` module load breaks); shared `runOptimize` in `optimizeHandler.ts`; capture/inject stay no-op in browser. Shell is Windows PowerShell — no `&&` or heredocs; chain with `;`, multi-line git commits via multiple `-m` flags.
- API keys stored encrypted via OS Credential Manager (`keyStore`); `orchestrator.ts` requires key and throws on missing key, API failure, or empty rewrite — no `optimizeLocal` fallback; streams chunks live to overlay/Studio (both show optimize errors). `localOptimizer.ts` kept for unit tests. Overlay output is plain refined prompt text (`stripResponseArtifacts`). UIA snapshot metadata stays local for Apply — only prompt text goes to the rewrite API.
- Overlay UI: `positionOverlayCenter()` + renderer `items-center justify-center`; transparent window (no scrim), IPC drag, `overlayPosition` persisted. Shell is `div.apple-glass` in `Overlay.tsx` — layered CSS gradients (~90% alpha) with `::before`/`::after` highlights in `index.css`; `GlassSurface` removed. CSS `backdrop-filter` cannot blur the OS desktop through a frameless transparent Electron window on Windows — frosted look is simulated via opaque gradients/tint only; do not use acrylic on the overlay window. Output uses `useTypewriterReveal` during Generate/Regenerate (~3 chars/frame with catch-up; `prefers-reduced-motion` skips animation) and native `scroll-thin` scrollbar. `ModelPicker` measures label width for chevron placement; `LevelSlider` beside Generate; footer order Discard → Apply → Copy (terminal: Discard → Copy only). `terminalContext` empty capture shows "Select text in the terminal first, then press the hotkey."
