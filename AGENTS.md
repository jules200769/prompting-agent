# AGENTS.md

## Learned User Preferences

- Dutch-speaking user; respond in Dutch when they write in Dutch. App UI is English-only (overlay placeholders, status strings, error messages).
- Target model picker selects guide methodology only; settings expose a single OpenAI API key; one rewrite model (GPT-4.1) handles all generation. Do not inject style-matching `styleHint` into the rewrite — user removed it because it interrupts/overrides the request.
- Ctrl+Shift+O overlay must auto-populate the draft from the active Windows text field — including when typing inside PromptForge Studio, not only external apps like Cursor/Chrome; manual paste of the draft is not acceptable. Non-terminal highlighted/selected text must NOT fill the overlay draft — open empty compose instead; empty mode must not carry "selected passage" rewrite instructions in `captureContext`. Apply must inject back into the captured field (Cursor/Chrome/native); on inject failure refined text must stay on the clipboard for manual paste — never restore the pre-capture clipboard over the fallback copy. Chat/transcript grounding is separate: pin/paste Session Context inside the already-open overlay (composer focused for the hotkey) — never hotkey-capture while a Cursor transcript is focused; rejected external transcript-selection as a UX failure that stalls or breaks the core flow. Import context modal offers Session|Project scope (`ContextImportScope` / `contextImportPromptFor`) — same UI flow, project prompt targets project-level context instead of session. Session vs project stay separate; `ContextBar` opens `ContextPanel` for resume/new-session-in-project/+ Project/Bring context/edit memory/trash (not the ⋮ menu). First Refine sets session title via `deriveSessionTitleFromPrompt`. Session rows and ContextBar dot use linked project accent (`assignProjectColor` / `PROJECT_COLOR_PALETTE`). Copying the export-context prompt sets `resumeContextModalRef` so the next hotkey reopens Import-context modal. Deleting a project warns and cascades sessions linked via `projectId`.
- Overlay does not auto-optimize on open; Refine button (or Enter when idle) triggers the rewrite API.
- Rejected clean-only conservative rewrite; refinement must follow model-specific prompting-guides/ with level-specific structure contracts (Cool plain prose through Max full XML). Never add examples, few-shot samples, or illustrative sample content at any level (including Max — success criteria only). Available destination/session/project context never excuses dropping or softening the selected model+level guide rules.
- Prefer clear API errors over silent local/mock fallback when rewrite fails — no substitute rewrite output on missing key or provider errors.
- Overlay output should be plain refined prompt text only — no JSON, no rubric/score UI in the overlay; after generation the output textarea is editable and Apply/Copy use the edited text; Generate becomes Regenerate after the first run in a session (resets on new capture).
- PromptForge optimizes prompts for the target LLM; launch is overlay/tray-only until user opens Studio via Settings or tray; RAM/geheugenoptimalisatie mag UX niet wijzigen — overlay hotkey, capture, Refine en Apply blijven identiek.
- Overlay opens on the active display at the user's snap placement (default center; five corners+center via ⋮-menu skeleton, `overlayPlacement` persisted, applied on hotkey open and when changed — no free drag); no black scrim/backdrop (including Import context modal — no `bg-black/50`); no open animation or flashy flicker — speed and subtle modern look prioritized. Backdrop pointer-down dismisses overlay (`onBackdropPointerDown`, `px-4 py-4` — import modal blocks, context panel closes first). Capture-wait spectral border glow during native capture in flight — visual only, never change capture orchestration/timing; glow ends instantly when capture completes (no fade-out); output placeholder during capture shows `contextStatusPlaceholder`, not "Capturing…". Hotkey UX: glass shell instantly (`showOverlayShell`); captured text fills shortly after; no `OVERLAY_CAPTURE_PENDING` spinner; first hotkey after reload is gold-standard. Empty capture: no yellow `text-warn` banner — open normal empty compose; do NOT auto-focus the Original prompt textarea when `mode === "empty"` (no caret on empty hotkey open). `hideForCapture` must NOT opacity-dim the already-visible overlay shell (dims+restore looks like a double popup on empty).
- Overlay shell is CSS glassmorphism on `.apple-glass` only — no React Bits `GlassSurface`, no Windows `setBackgroundMaterial('acrylic')` on the full overlay window (causes large white box); inner `apple-glass-panel`/`apple-glass-pill`/`apple-glass-menu` unchanged; pill buttons less transparent than shell.
- Overlay footer UX: L1–L4 draggable level slider beside Generate — thin grey track, colorful Cool/Warm/Hot/Max label, spring snap; Discard next to Apply/Copy in light gray (`text-white/45`), not pink; terminal sessions use the same footer (Apply injects, not copy-only). Terminal rewrite output must be one line — no Enter/newlines (shell paste breaks); overlay blocks Enter in output textarea.
- Overlay chrome: only left decorative "PromptForge" wordmark folder tab remains (`.apple-glass-tab--left`, z-0 under card). `ModelPicker` lives in footer (not a bottom tab). Overflow menu = three horizontal lines + "menu" label under Copy — shell sibling at z-40 via `copyAnchorRef`/`menuPos`, outside `.apple-glass` overflow:hidden; dropdown pops UP (`bottom-full mb-1`); must not expand modal/footer or shift Discard/Apply/Copy. Capture-wait glow uses existing accent palette only (#7c6cff, #9d92ff, cyan 105,214,255) — brighten alphas, no new hues. `promptType` still suffixes opt-cache hash with `|type:{t}` but PromptTypePicker UI was removed.

## Learned Workspace Facts

- PromptForge: Windows-only Electron + Vite + React/TypeScript; npm scripts `dev`, `build`, `test` (vitest), `lint`; architecture in `src/main`, `src/renderer`, `src/engine`, `src/shared`. Dev bundler: `vite.config.mts` with `@vitejs/plugin-react-swc`, `css.devSourcemap: false`, `hmr.overlay: false`; Studio lazy-loaded in `router.tsx` (separate chunk).
- Rewrite always uses `REWRITE_CONFIG` (OpenAI `gpt-4.1`); target model picker selects which `prompting-guides/` file `guideLoader` loads.
- `prompting-guides/` source of truth per ModelId: `opus4.8.md`, `gpt5.5.md`, `gemini.3.pro.md`, `deepseek.V3.md`, `grok4.md`, `composer2.5.md`.
- L1–L4 = guide structure adherence scale (Cool/Warm/Hot/Max); knob sets guide excerpt + rewrite intensity. `getLevelStructureContract()` in `guideLoader.ts` fixes output shape: Cool=plain prose; Warm=`<instructions>`+`<input>`; Hot=`<context>`/`<task>`/`<constraints>`/`<output_format>`; Max=Hot plus `<success_criteria>` only (never `<examples>` or illustrative samples; must retain all L3 tags). `buildMetaPrompt` in `providers.ts` embeds contract plus action/constraint framing from L2+. `REWRITE_PIPELINE_VERSION` in cache hash; Regenerate sends `skipCache: true`. Temperature fixed at `REWRITE_CONFIG.temperature` (0.3); measured adherence via `adherenceLevel()` in `rubric.ts`.
- Launch: tray-only — no Studio window on dev/launch (removed `ensureStudio()` from startup and `app.on("activate")`); overlay on Ctrl+Shift+O; Studio opens only via the overlay Settings button or tray, both routed through `STUDIO_SETTINGS`/`ensureStudio("settings")` (tray Settings was previously broken — `webContents.send` lacked a preload bridge).
- Hotkey flow: `prepareCaptureTarget()` → `showOverlayShell()` (instant glass, empty field, `setOpacity(1)`) → `hotkeySnapshot()` → fast path skips `hideForCapture()` → `captureSelection()` → `deliverCaptureToOverlay()` fills captured text (second `OVERLAY_SHOW`; no `OVERLAY_CAPTURE_PENDING` on hotkey). Empty field with UIA uses the compose fast path (skip `hideForCapture` / `win-capture.ps1`) so the shell does not dim and pop back. **`hideForCapture` must never opacity-dim the already-visible overlay** — only hide other PromptForge windows. **Do not reorder to capture-before-shell** — that delays visible popup until PS capture completes even on fast path (regressed in a later session). **Blur hide must skip while `hotkeyInFlight`** — otherwise `showOverlayShell` + snapshot `FocusWindow` closes the overlay mid-capture and popup reappears only after `deliverCaptureToOverlay` (feels slow). Resident overlay: `hideOverlay()` sends `OVERLAY_CLEAR`, awaits `OVERLAY_PREPARED`, sets opacity 0 — does not `hide()`; `overlaySessionOpen` drives blur/toggle (not `isVisible()`). `revealOverlay()` idempotent: `setOpacity(0)` → `showInactive()` only if not visible. `primeOverlayBuffer()` at startup leaves window warm via `showInactive()`. Overlay `backgroundThrottling: false`; renderer `ackOverlayPrepared()` 1× rAF then main `setOpacity(1)` on shell reveal (Windows transparent-window white-flash guard only, not an open animation). Reset Import-context modal state (`contextModalOpen` / import fields) on overlay clear/show so a later hotkey never reopens the context modal.
- PS script shared types: `terminal-io.ps1` OWNS the shared `WinFg` C# class; scripts that dot-source it (`win-hotkey-snapshot.ps1`, `win-inject.ps1`) must NOT define their own `WinFg` unguarded — a duplicate `Add-Type` throws `TYPE_ALREADY_EXISTS` under `$ErrorActionPreference='Stop'`, which crashes `hotkeySnapshot` and forces the slow `win-capture.ps1` fallback (laggy popup that appears then reopens). `win-hotkey-snapshot.ps1` keeps its local `WinFg` behind `if (-not ([PSTypeName]'WinFg').Type)`. `win-inject.ps1` uses `WinInject`, `win-capture.ps1` uses `WinCapture` (distinct names, no collision); `win-hotkey-snapshot.ps1` also owns the context-layer `PfCtx` class (GetWindowText) — guarded like the rest, never folded into `WinFg`. `WinFg.GetWindowThreadProcessId` P/Invoke must be `out uint lpdwProcessId` (not `IntPtr`); never use `$pid` as out var — PowerShell `$PID` is read-only; use `$winPid`.
- Capture/inject via PS scripts (`win-capture.ps1`, `win-hotkey-snapshot.ps1`, `win-inject.ps1`); foreground HWND polled every 150ms via koffi `src/main/win32.ts` (`GetForegroundWindow`, not `get-foreground.ps1` per tick); `pollInFlight` mutex; `ps-warmup.ps1` + `warmCaptureBridge()` at launch reduce first-hotkey PS/UIA cold start. Shared terminal PS helpers in `scripts/terminal-io.ps1` (`Resolve-TerminalPaneElement`, `Focus-TerminalPane`, keyboard/clipboard). Terminal hosts use `CaptureMode` `"terminal"`: hotkey snapshot is **UIA-only** (`TextPattern.GetSelection()` / document text) — do NOT revive `Invoke-TerminalPromptLineCopy` / `Invoke-TerminalSelectionCopy` keyboard clipboard fallbacks in `win-hotkey-snapshot.ps1` (they inject real Ctrl+C/`^C` into focused shells and break `npm run dev`). `win-capture.ps1` is selection-only clipboard grab — no Ctrl+C-before-select-all fallback. Frozen `terminalBounds` from capture flows to inject when UIA resolve fails. `terminalDetect.ts` detects native class/process plus Cursor/VS Code integrated panes (`focusedIsTerminalPane`); filters a11y noise/window-title leaks. Gotcha: Cursor chat transcript focus can be misclassified as an integrated terminal with no selection, stalling capture ~30s and returning empty — do not revive hotkey-on-transcript workflows. All paths normalize via `resolveCaptureFromPossibleTerminal()`/`extractTerminalInput()` in `terminalCapture.ts` — keeps scrollback output + current prompt line, strips banners/a11y noise. Text-shape heuristics (`isLikelyFullConsoleBuffer` / `isTerminalBufferDump`) are a missed-terminal fallback only — never override known non-terminal controls (`isKnownNonTerminalControl`: aislash/monaco/ProseMirror/…) or `focusedIsTerminalPane === false`. Cursor `@`-mention chips expose Codicon BMP Private Use Area glyphs (`U+E000`–`U+F8FF`) in UIA/clipboard text (tofu without Codicon) — often quote-wrapped multiline chrome like `''\uE099\nfilename\n\uEA76''`; `stripUiPrivateUseGlyphs` in `captureTextSanitize.ts` (via `sanitizeCaptureText`/`finalizeCaptureText` and `contextLayer` sidecar fields) strips PUA + glued quote chrome; UIA usually yields the display name only, not the full `@path`.
- Apply: `injectText()` via `win-inject.ps1` using frozen target from capture (`injectStrategy.ts` hostKind native/chromium/richEditor/terminal; `scripts/inject-strategy.ps1` in PS scripts); overlay label "Apply" for all hosts including terminal. Field strategy: one focus-restore → resolve element → host-specific primary+fallback (native: ValuePattern→clipboard paste→unicode; chromium/richEditor: clipboard paste→unicode) — not the old 5-method cascade. Terminal: `freezeTerminalInjectTarget()` on capture; same IPC path as fields (no `main.ts` terminal clipboard short-circuit); `Try-TerminalPaste` = single focus+select+paste with optimistic verify — right-click paste WT/conhost, Ctrl+V integrated; no strategy waterfall or clipboard readback verify cascade. `main.ts` `waitUntilForeground()` + ~200ms settle before inject; failure `revealOverlayAfterInjectFallback()` keeps session, success `finalizeOverlayAfterInject()`. Clipboard fallback on inject failure keeps refined text; restore pre-capture snapshot only after verified inject (~200ms). Ctrl+A/V via `keybd_event` (SendInput unreliable for combos from PS x64); Unicode typing still SendInput. Chromium/richEditor: optimistic verify when UIA readback fails after paste. `terminalContext` on optimize → TERMINAL SHELL meta-prompt (single-line plain output overrides L2+ tags); `toTerminalSingleLine()`/`stripTerminalStreamChunk()` in `terminalOutput.ts` on stream/Apply/inject; cache hash includes `|terminal`. PS scripts read fresh per hotkey — script edits need no rebuild; TS changes need `npm test`/dev reload. Overlay "Original" is editable `<textarea>`; readOnly while busy/`"capturing"`; Enter inserts newline (does not trigger Refine). Preload registers `OVERLAY_SHOW`/`onOverlayCapturePending` with replay; `IPC.OVERLAY_CLEAR` on hide; overlay created hidden; `deliverCaptureToOverlay()` waits for `did-finish-load` if needed.
- Dev/memory: `optCache` LRU cap 100 (slim entries, no persisted `diff`); `spellcheck: false`; overlay `backgroundThrottling: false`, Studio `true`; Studio nulled on `closed`; no React StrictMode in dev. Run ledger: each optimize → `RunRecord` in storage + append-only `userData/run-history.jsonl` (`runHistoryJsonl.ts`); Studio history UI with per-run comments; overlay `historyFinalize` on Apply/Copy. Do not auto-open detached DevTools (steals focus, breaks capture); browser UI at `localhost:5173#/overlay-preview` only. Vite proxies `/api/optimize` + `/api/settings` to `127.0.0.1:5174` — never all `/api`. Capture/inject no-op in browser. PS shell: chain with `;`, no `&&`/heredocs. `dev:electron` runs `tsc -p tsconfig.electron.json` once then launches Electron — Vite HMR does not rebuild main/shared; after `src/main` or `src/shared` changes, restart `npm run dev` (or rebuild main) or the running app keeps stale capture/sanitize code. `tsconfig.electron.json` must `"exclude": ["**/*.test.ts"]` — Vitest files (esp. `contextLayer.test.ts` with top-level `await`/`vi.mock`) must not compile into the Electron CommonJS build or `npm run dev` fails; `tsconfig.scripts.json` + `scripts/tsconfig.json` typecheck Node scripts (`calibration-round.mts`).
- API keys stored encrypted via OS Credential Manager (`keyStore`); `orchestrator.ts` requires key and throws on missing key, API failure, or empty rewrite — no `optimizeLocal` fallback; streams chunks live to overlay/Studio (both show optimize errors). `localOptimizer.ts` kept for unit tests. Overlay output is plain refined prompt text (`stripResponseArtifacts`). Context layer (keep kinds separate): (1) Destination — with `screenContext` on (default), `assembleCaptureContext()` in `contextLayer.ts` during the hotkey snapshot pass sends app identity (process, window title ≤200, hostKind, browser site), capped selection structure (selectedText ≤4000, beforeCursor tail ≤1500, afterCursor head ≤500), and matched editor file names (≤10, from `fileMemory`) as a DESTINATION CONTEXT block in `buildMetaPrompt` (`contextSignals.ts`).
