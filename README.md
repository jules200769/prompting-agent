# Anvyll — MVP

**Product:** Anvyll · **Domain:** [anvyll.app](https://anvyll.app) (not purchased yet)

A Windows-native AI prompt optimization studio. Write a rough prompt anywhere,
press a global hotkey, and Anvyll rewrites it according to the **official
prompt-engineering methodology of the specific target model** (Claude Opus 4.8,
GPT-5, Gemini 3, DeepSeek V3, Grok 4), then applies the optimized prompt back
into the original text box — or copies it to the clipboard.

This is the working MVP described in `CONCEPT.md`. It implements the two-axis
thesis: **model-specific engines as data** + a **Windows-native system-wide
hotkey overlay**.

## What's in the MVP

- **Global hotkey overlay** (`Ctrl+Shift+O` by default, reassignable).
  - Hotkey → snapshot clipboard → restore focus to the app you were in →
    simulate `Ctrl+C` (selection) or `Ctrl+A`+`Ctrl+C` (full text field) →
    show the overlay with the captured text.
  - If capture fails, the overlay shows an error — no manual paste required.
  - **Apply** restores focus, replaces the field content (UI Automation or `Ctrl+A` + `Ctrl+V`), then restores your original
    clipboard. **Copy** leaves the optimized prompt on the clipboard.
  - Clipboard snapshot/restore so we never clobber what you copied.
- **Target model vs rewrite model** — you pick the **target model** (Claude Opus 4.8,
  GPT-5, Gemini 3, …) to select that model's prompt-engineering methodology.
  All rewrites are generated by a single **GPT-4.1 mini** call via OpenAI; the
  target-model pack supplies the expertise — you do not need an API key for each
  provider.
- **Model-specific prompt-packs as data** (`src/engine/packs.ts`) — not
  hardcoded in the client. Each pack carries the provider's methodology, an
  exemplar, the rubric weights, and per-level overrides (L1–L4).
- **Optimization engine** (`src/engine/`): heuristic 1–100 rubric analyzer
  (Appendix A), meta-prompt assembly from the selected pack, streaming, and a
  line-oriented diff with semantic tags (`+ Role/persona`, `+ Constraints`, …).
- **Rewrite provider** (BYOK): one **OpenAI API key** drives all LLM rewrites
  through GPT-4.1 mini, regardless of target model.
- **Local fallback optimizer**: with no API key, the app still produces a real,
  structured, model-specific prompt (XML-tagged for Claude, markdown headers for
  GPT/Gemini/DeepSeek/Grok). The free/offline tier works with zero config.
- **Studio** (full workbench): Original | Optimized diff view, analysis panel
  with score ring + rubric chips + weaknesses, persona suggestion, library,
  history, and settings (hotkey, defaults, persona/context memory, BYOK keys).
- **Secure key storage** via Electron `safeStorage` (Windows Credential Manager).
  Keys never leave the machine; only presence flags are exposed to the UI.
- **Local SQLite** store for settings, library, history, and an identical-input
  optimization cache (dedup for repeat prompts).

## Architecture

```
src/
  main/        Electron main: windows, tray, hotkey, IPC, capture/inject (Win32 via PowerShell SendInput)
  preload/     Allowlist IPC bridge (contextBridge)
  engine/      packs.ts (engines-as-data), rubric.ts, diff.ts, providers.ts, orchestrator.ts, localOptimizer.ts
  renderer/    React + Tailwind: Overlay + Studio
  shared/      Types + IPC channel names (shared between main/preload/renderer)
```

The concept specifies Tauri 2.0 (Rust). This MVP uses **Electron** instead —
functionally equivalent for the Windows-only MVP (global hotkey, clipboard
capture/inject via Win32 `SendInput`, secure key storage, local SQLite) — because
the target machine has Node but no Rust toolchain. The engine, packs, IPC
contract, and UX are framework-agnostic and port directly to Tauri later.

## Run it

Requirements: **Node 18+ and npm** (Windows). Electron downloads its own
binary; no native build toolchain is required except `better-sqlite3`, which
ships prebuilt binaries for Windows.

```powershell
npm install
npm run dev
```

`npm run dev` starts the Vite dev server (port 5173) and launches Electron
against it with hot reload. The Studio window opens on first launch; the overlay
is summoned with the global hotkey.

### Build a distributable

```powershell
npm run build
npm start            # run the built app
npx electron-builder --win   # produce an installer in /release
```

## Using it

1. **Open Studio** (opens on launch, or from the tray). Go to **Settings**.
2. (Optional) Add your **OpenAI API key** for LLM-quality rewrites via GPT-4.1 mini.
   Without a key, Anvyll uses the local fallback optimizer.
3. Set your **default target model/level** and (optional) **persona / context memory**.
4. Anywhere on Windows — ChatGPT Desktop, Cursor, a browser, Word, Notion, a
   terminal — place the cursor in a text field with your rough prompt and press **Ctrl+Shift+O**.
   Selected text is used if present; otherwise the entire field is captured automatically.
5. The overlay appears with the streamed optimized prompt, the quality score,
   and the diff. Press **Enter** to apply, or click **Copy**.

## Keyboard (overlay)

- `1`–`4` — set optimization level
- `Enter` — apply (when done)
- `Esc` — dismiss
- `Tab` — cycle focus

## Notes & limitations (MVP scope)

- Levels 3–4 (expert/maximum with self-critique) are wired in the packs and
  meta-prompt but the LLM self-critique loop runs in a single streamed pass in
  this MVP (the prompt instructs critique-then-refine); a multi-call pipeline is
  a Phase 2 item.
- Cloud sync, billing, AI Research Mode, Team plan, and the full Studio
  before/after answer comparison are deliberately out of MVP scope (see
  `CONCEPT.md` §10).
- Synthetic input can be blocked by some secure fields; the overlay always
  offers **Copy** as a fallback so we never fail silently.
